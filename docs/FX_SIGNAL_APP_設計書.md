## 1. プロジェクト概要

| 項目 | 内容 |
|---|---|
| アプリ名 | FXナビ(仮) |
| 目的 | FX初心者である利用者が「今すぐ分析」ボタンを押した時点の相場を、多数のテクニカル指標で総合分析し、**売買判定・エントリー価格・指値(利確)・逆指値(損切り)**を初心者にもわかる日本語で提示する |

- ファンダメンタルズ分析(ニュース内容の解釈・良し悪しの予想)はしない。経済指標カレンダーは「**いつ危険な時間帯が来るか**」の検知にのみ使う(§7.4)
- ユーザー登録・サーバーサイド処理

### 1.2 免責表示(必須実装)

画面フッターと初回起動時のダイアログに以下を表示すること:

### 3.1 データソース(優先順)

| 優先 | ソース | 内容 | キー | 備考 |

| 優先 | ソース | 内容 | キー | 備考 |
|---|---|---|---|---|
| 1 | **Forex Factory 週間カレンダーJSON**<br>`https://nfs.faireconomy.media/ff_calendar_thisweek.json` | 今週の経済指標(日時・通貨・重要度High/Medium/Low・指標名) | 不要 | 時刻はUTC→JSTに変換して表示。指標名は英語なので、主要指標(下記)は日本語名の対訳辞書を内蔵すること |

- 優先度1をまず試し、CORSエラー・通信失敗時は自動で次にフォールバックすること

- キャッシュは「確定済みの過去データ」(過去の日・年のローソク足)のみ恒久的に使い、**分析ボタン押下時には当日分・当年分・現在値のキャッシュを破棄して必ず再取得すること**(押した瞬間の値を反映するため。API制限対策と鮮度の両立)

`USD/JPY`(初期選択)、`EUR/JPY`、`GBP/JPY`、`AUD/JPY`、`EUR/USD`、`GBP/USD`、`AUD/USD`

## 4. 用語と共通定義

- **OHLC**: 各ローソク足の始値(Open)・高値(High)・安値(Low)・終値(Close)
- **確定足**: 現在進行中の足は判定に使わず、**直近の確定した足**でシグナル判定すること(だまし防止)
- **pips**: クロス円は 0.01円 = 1pip、それ以外は 0.0001 = 1pip
- 各指標のシグナル値は `+1`(買い)/ `0`(中立)/ `-1`(売り)の3値。連続値を返せる指標は −1〜+1 の小数も可

`text` は初心者向けの一言解説(§8.4 の文例に従う)。

### 5.5 計算式の指定(あいまいさ排除のため)

- **SMA**: 終値の単純平均
- **EMA**: `EMA_t = 価格_t × k + EMA_{t-1} × (1−k)`、`k = 2/(期間+1)`、初期値は最初の期間分のSMA
- **RSI**: Wilder方式(平滑移動平均)。`RSI = 100 − 100/(1+RS)`、`RS = 平均上昇幅/平均下落幅`
- **MACD**: `MACD線 = EMA12 − EMA26`、`シグナル線 = MACD線のEMA9`、`ヒストグラム = MACD線 − シグナル線`
- **ATR**: `TR = max(高値−安値, |高値−前終値|, |安値−前終値|)` のWilder平滑14期間
- **ADX**: Wilder方式。+DM/−DM→+DI/−DI→DX→ADX(14期間平滑)
- **RCI**: 日付順位と価格順位のスピアマン順位相関 ×100
- **ピボット**: `P=(H+L+C)/3`、`R1=2P−L`、`S1=2P−H`、`R2=P+(H−L)`、`S2=P−(H−L)`、`R3=H+2(P−L)`、`S3=L−2(H−P)`
- **CCI**: `(TP − TPのSMA20) / (0.015 × 平均偏差)`、`TP=(H+L+C)/3`
- **ストキャス**: `%K = (C−最安値14)/(最高値14−最安値14)×100` をスロー化(3期間SMA×2段)

## 6. 総合判定エンジン

### 6.1 スコアリング

1. **時間足ごと**に全指標のシグナル(−1〜+1)× 重みを合計し、重み合計で割って正規化 → 時間足スコア(−100〜+100)
2. 時間足スコアを以下の比率で加重平均 → **総合スコア**(9時間足)

| 時間足 | 比率 |
|---|---|
| 1分足 | 5% |
| 5分足 | 8% |
| 15分足 | 10% |
| 30分足 | 12% |
| 1時間足 | 20% |
| 4時間足 | 15% |
| 8時間足 | 10% |
| 日足 | 12% |
| 週足 | 8% |

※データ不足で除外した時間足がある場合は、その比率を残りに按分して合計100%を維持すること。

3. 総合スコアを5段階判定に変換:

| 総合スコア | 判定 | 表示色 |
|---|---|---|
| +40以上 | 🔴 強い買い | 濃い赤(日本の株価配色: 上昇=赤) |
| +15〜+40 | 買い | 赤 |
| −15〜+15 | ⬜ 中立(様子見) | グレー |
| −40〜−15 | 売り | 青 |
| −40以下 | 🔵 強い売り | 濃い青 |

### 6.2 相場レジーム判定(重みの動的切替)

1時間足の ADX で相場の状態を判定し、指標の重みを切り替えること:

| 状態 | 条件 | 重み補正 |
|---|---|---|
| トレンド相場 | ADX ≧ 25 | トレンド系(#1〜6)の重み ×1.5、オシレーター系(#8〜13)×0.5 |
| レンジ相場 | ADX < 20 | オシレーター系 ×1.5、トレンド系 ×0.5、ボリンジャー逆張りモード |
| 移行期 | 20 ≦ ADX < 25 | 補正なし |

判定結果には「いまは**トレンド相場**なので、流れについていく指標を重視しています」のような説明文を必ず添えること。

### 6.3 矛盾・注意の検出

以下を検出したら判定カードに ⚠️ 注意文を表示すること:

- 時間足どうしで方向が逆(例: 1時間足は買い、日足は売り)→「短期は買い優勢ですが長期は下落トレンドです。逆張りになるため慎重に」
- 総合スコアが中立圏 →「明確なシグナルがありません。**取引を見送るのも立派な戦略です**」
- スクイーズ検出(#16)→「値動きが極端に小さくなっています。急変に注意」
- 重要指標(週末・早朝の流動性低下): 土日はFX市場が閉まっている旨を表示し、金曜NY引け後は「週明けの窓開きリスク」を警告

## 7. 売買プラン算出(エントリー・指値・逆指値)

売買プランは §7.3 の厳選フィルタをすべて通過したときのみ提示する。通過しない場合は「エントリー見送り」と理由を明示。
以下は買いの場合(売りは対称に反転)。ATRは1時間足の値を使う。

### 7.1 スイング高値・安値の検出

- スイングハイ: 左右5本のどの高値よりも高い足の高値
- スイングロー: 左右5本のどの安値よりも安い足の安値
- 直近サポート = 現在値より下で最も近いスイングロー(±0.2×ATR以内の複数点はクラスタとして平均)
- 直近レジスタンス = 現在値より上で最も近いスイングハイ(同様)

### 7.2 価格の算出式

| 項目 | 算出式 | 説明文の例 |
|---|---|---|
| エントリー価格 | 原則: 現在値(成行)。ただし現在値が直近サポートから 1×ATR 以上離れている場合は「サポート+0.3×ATR への押し目買い指値」を提案 | 「今すぐ買う場合は◯◯円。少し待てるなら◯◯円まで下がったところで買う指値がおすすめ」 |
| 逆指値(損切り) | 直近サポートが**2×ATR以内**にあれば `サポート − 0.3×ATR`、なければ `エントリー − 1.5×ATR`(※サポートが遠いときに損切り幅が広がりすぎて利確目標が非現実的になるのを防ぐ) | 「予想が外れたときに損を確定させる保険。◯◯円に必ず入れてください」 |
| 指値(利確) | 第1目標: `エントリー + 1.5×リスク幅`(リスク幅=エントリー−逆指値)。第2目標: 直近レジスタンス手前(−0.2×ATR)。**遠い方は表示のみ、近い方を推奨値とする** | 「利益を確定する予約注文。◯◯円に指値を置きます」 |
| リスクリワード比(RR) | `(指値−エントリー)/(エントリー−逆指値)` | RRも画面に表示 |
| **RRフィルタ** | RR < 1.2 の場合、判定を強制的に「見送り」に変更し理由を表示 | 「勝ったときの利益より負けたときの損の方が大きい位置なので、今回は見送りです」 |

### 7.3 厳選フィルタ(勝率を優先し、怪しい局面はすべて「見送り」にする)

エントリー提案は**条件が揃ったときだけ**出すこと。以下のいずれかに該当したら、総合スコアにかかわらず判定を「見送り」にし、理由を表示する:

| フィルタ | 見送り条件 | 表示する理由の例 |
|---|---|---|
| RRフィルタ(§7.2) | RR < 1.2 | 「負けたときの損の方が大きい位置です」 |
| 上位足一致フィルタ | 買い提案なのに 4時間足・8時間足・日足のうち**2本以上**のスコアが −15 未満(売り方向)。売り提案は対称。**1本だけの逆行は見送りにせず注意文の表示にとどめる** | 「長い時間足の複数が逆方向です。逆張りは勝率が下がるので見送ります」 |
| 確信度フィルタ | 総合スコアの絶対値が 15 未満(判定が「中立」) | 「判定が中立のため見送ります。チャンスは何度でも来ます」 |
| 高ボラ警戒 | 直近3本のいずれかで値幅が 3×ATR 超(急変動直後) | 「急変動の直後は値が飛びやすいため見送ります」 |
| 重要指標フィルタ(§7.4) | 選択通貨ペアに関係する重要度「高」の経済指標の **発表1時間前〜発表30分後** | 「まもなく米雇用統計の発表です。発表前後はチャート分析が通用しない値動きになるため見送ります」 |

※「取引回数を増やすこと」ではなく「負けにくい局面だけを提示すること」をこのアプリの最優先方針とする。見送り表示の頻度が高くても仕様どおりの正しい動作である。

**見送り時の参考プラン(必須):** 見送り判定のときも、「それでもエントリーするなら」という折りたたみセクションに、通常時と同じ算出式(§7.2)によるエントリー価格・逆指値・指値・リスクリワード比を**参考データとして併記**すること。方向は総合スコアが傾いている側とする。「非推奨」であること、逆指値を必ず入れることを明示し、チャートのプラン線にも「(参考)」と表示すること。

### 7.4 経済指標カレンダー連携(急変動の予報)

テクニカル分析が最も裏切られるのは重要経済指標の発表前後である。カレンダー(§3.1)を使って以下を実装すること:

| 機能 | 仕様 |
|---|---|
| 対象の絞り込み | 選択通貨ペアの構成通貨(例: USD/JPY → USD と JPY)に関する指標のみを対象とする |
| 見送り判定 | 重要度「高」の指標の発表1時間前〜30分後は §7.3 のフィルタで「見送り」 |
| 事前警告 | 重要度「高」の指標が **12時間以内**にある場合、判定は出すが ⚠️「◯時◯分に◯◯(重要度: 高)の発表があります。それまでにポジションを閉じるか、逆指値を必ず入れておいてください」と警告 |
| 今後の予定表示 | 「今後24時間の経済指標」リストをUIに表示(§8.1)。JST変換した時刻・日本語名・対象通貨・重要度バッジ(高=赤/中=黄/低=グレー) |
| 初心者向け説明 | 用語集に「経済指標」「雇用統計」「FOMC」「日銀金融政策決定会合」を追加し、「発表の瞬間は1分で1円動くこともある」ことを説明 |

### 7.5 時間帯の知識(市場セッション)

APIを使わず端末の時計だけで判定できる、値動きの「時間帯のクセ」も注意表示すること:

| 時間帯(JST) | 表示 |
|---|---|
| 平日 5:00〜8:00 | ⚠️「早朝はスプレッド(売値と買値の差)が広がりやすく不利な時間帯です」 |
| 8:00〜15:00 | 「東京時間: 比較的おだやかな値動きの時間帯」 |
| 16:00〜21:00 | 「ロンドン時間: 値動きが活発になる時間帯」 |
| 21:00〜翌2:00 | 「NY時間: 最も取引が活発。米国の経済指標もこの時間帯に集中」 |
| 金曜 NY引け間際・土日 | 既存の週末警告(§6.3)に統合 |

現在どのセッションかをヘッダー付近にバッジ表示すること。

### 8.4 ステルスモード(擬態画面)

周囲に画面を見られてもFX取引中だとわからないようにする機能。

- 表示内容: 市役所の「国民健康保険の各種手続きについて」風の案内ページ(全面・常にライト配色)。**タブのタイトルとファビコンも同時に差し替える**こと
- 直近の分析結果をページ末尾の「文書管理番号」にさりげなく埋め込む:
  `文書管理番号: 2026-{現在値の数字6桁}-{B|S|N}{強さ0〜100}(関連様式: 第{逆指値の数字}号・第{指値の数字}号)`
  B=買い / S=売り / N=中立。数字は総合スコアの絶対値(40以上=「強い」)
- 擬態ページ内に「窓口区分」セレクト(=通貨ペアの偽装。UJ/EJ/GJ/AJ/EU/GU/AU)と
  「最新の情報に更新」ボタン(=擬態したまま再分析。デモ確認ダイアログは出さない)を置くこと
- 解除: Escキー、右上の「印刷用表示」リンク、または文書管理番号のダブルタップ
- コードの読み方は用語集に収録すること

### 8.5 初心者向け解説の文体ルール(全UIテキスト共通)

- 専門用語には必ず一言説明を付ける。例: 「RSI(買われすぎ・売られすぎを0〜100で表す体温計のような指標)は現在 28 で、**売られすぎ**の水準です」
- 断定しない。「〜の傾向があります」「〜が期待できます」
- 判定理由は必ず自然な文章で。例: 「1時間足では移動平均線が上向きに揃い、MACDも買い転換したため、上昇の流れが出ています。ただし日足ではまだ下落トレンドのため、短期の戻りを狙う形になります」
- 用語集モーダルに全20指標+「指値」「逆指値」「pips」「スプレッド」「レバレッジ」「リスクリワード比」「損切り」「経済指標」「雇用統計」「FOMC」「日銀金融政策決定会合」の解説を収録すること

## 10. 非機能要件

- 分析ボタン押下から結果表示まで **5秒以内**(API応答を除く計算時間は500ms以内)
- 指標計算関数はすべて純粋関数とし、ファイル内に簡易セルフテスト(既知のOHLC入力→期待値)を `?test=1` クエリで実行できるようにすること(最低: SMA, EMA, RSI, MACD, ATR, ピボットの6つに既知値テスト)
- 外部送信はレート取得APIのみ。APIキー以外の個人情報は扱わない
- コードは日本語コメント付きで、指標ごとに関数を分離すること
